# Security Policy

APRICORE is a methodology specification and documentation site — there is no
runtime product or installable software, so the classic vulnerability surface
is minimal. Two things are still worth reporting privately rather than in a
public issue:

- **Documentation site** — a privacy, consent, or analytics problem on the
  published site (for example, tracking that runs without consent).
- **Build pipeline** — a misconfiguration in the GitHub Actions workflows.

## Reporting

Please report any such issue privately by email to **info@sebastian-mompour.de**
instead of opening a public issue. You can expect an acknowledgement within a
few days.

For ordinary documentation problems (typos, broken links, unclear wording),
please open a regular issue instead.
